Frequently Asked Questions

Quick answers to common questions about SF File Manager, security, and support

Security & Privacy

Is SF File Manager really secure?

Yes, SF File Manager uses military-grade AES-256-CBC and ChaCha20-Poly1305 encryption, the same standards used by governments, banks, and security agencies worldwide. Our implementation is FIPS 140-2 compliant and has been audited by security experts.

FIPS 140-2 Level 3 Certified
Zero-knowledge architecture

Can I recover my files if I forget the password?

No, we use zero-knowledge encryption, which means we cannot recover your password or decrypt your files. This is by design to ensure maximum security. We recommend using a secure password manager and creating backup copies of important passwords.

Password Security Tips:
  • Use a reputable password manager
  • Create strong, unique passwords
  • Store backup copies securely
  • Consider using passphrases

How does quantum-resistant encryption work?

Our quantum-resistant encryption uses post-quantum cryptographic algorithms that are designed to be secure against attacks from quantum computers. These algorithms are based on mathematical problems that even quantum computers cannot solve efficiently.

Features & Usage

Is SF File Manager free to use?

Yes, SF File Manager is completely free and open source. You can use all features without any limitations, hidden costs, or subscription fees. The source code is available on GitHub for transparency and community contributions.

No subscription fees
No feature limitations
Open source transparency

Which operating systems are supported?

Currently, we support Windows 10 and 11. macOS and Linux versions are in development and will be released in Q2 2025. All versions will have the same features and cross-platform compatibility.

Windows - Available Now
macOS - Coming Q2 2025
Linux - Coming Q2 2025

Can I encrypt entire folders and their subfolders?

Yes, SF File Manager supports recursive folder encryption. You can encrypt entire directory structures while preserving the folder hierarchy. The software will process all files and subfolders automatically with progress tracking.

Technical Support

How do I report a bug or security issue?

For general bugs, please use our contact form or create an issue on GitHub. For security vulnerabilities, please email us directly at security@sfmanager.com with details. We take security reports seriously and respond quickly.

General bugs: Contact form or GitHub
Security issues: security@sfmanager.com

What information should I include in a support request?

To help us assist you quickly, please include:

  • Your operating system and version
  • SF File Manager version number
  • Detailed description of the issue
  • Steps to reproduce the problem
  • Any error messages you received
  • Screenshots if applicable
